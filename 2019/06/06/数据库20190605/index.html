Power Designer能干什么？

截止到2019年6月，我所掌握的知识可以使用其中的：

  • 数据库：数据库建模工作、业务流程视图
  • UML图

总的来说，Power Designer可以进行建模工作，可以将你设计的图形化模型直接转化成代码。

数据库建模工作

概念模型建立

实体

  • 可以定义实体的属性
    • 属性名
    • 属性导出为代码时叫什么名字
    • 备注
    • 域：检查约束，用check实现

这里可以运行VB脚本：tools-Execute Commands

同时右键option可以更改样式

联系

  • 可以对两个实体建立简单的联系
    • 联系名
    • 一对多（父子关系）、多对一、一对一、多对多
    • 强弱

从院系看学生，至少0个，至多n个；从学生看院系，至少1个，至多1个

  • 但目前的联系只是一个对应属性，并没有存放联系的地方，无法表示关联实体
    • 建立一个新的实体
    • 把多对多联系转化成关联实体
    • 联系实体无需标识主键
    • “全部参与”的概念

可知，E-R图里没有外码

逻辑、物理模型建立

概念模型——逻辑模型——（sql server 或oracle可选）物理模型——（database-generate database)生成数据库脚本script/部署实施数据库(直接实施)

直接生成/间接生成可选

导出的时候会提供选项，选择代码生成的样式
